SOP Love Letter
  • ls - List information about file(s)
  • cd - Change Directory
  • mkdir - Create new folder(s)
  • .. - Previous directory
  • / - Root directory
  • touch - create empty file

Dear Bash,
I can't even list the things that I love about you. We've had so many changes and making so much history together. I sometimes wish that I could go back to the beginning so I can relive every moment again. My favorite thing to do in the morning when I wake up is to touch your face. Remember those nights when we would fall asleep to pingu? So silly but I was so happy. The more time I spend with you is the less I want to be away from you. You are my home and my master.

Sincerely, Terminal.

  • ping - Test a network connection
  • more - Display beginning of file
  • less - Display end of file
  • ~ - Home directory
  • sudo - Execute a command as another user
  • init - Create a new local repository
  • add - Add one or more files to staging (index)
  • clone - Checkout a local or remote repository
  • branch - List all the branches in your repo, and also tell you what branch you're currently in
  • commit - Commit changes to head (but not yet to the remote repository)

Dear git,
Initally, I never liked you, or thought that I would ever like you. I might add that never once I ever thought that you would fall in love with someone like me either. I was a clone of your mother whom you deeply despised. Commanding yet so high maintenance, like a stubborn branch that would never snap. Boy was I wrong. Once I saw what a committed person you were, I had no escape from you. You are the master of my heart I would never go back to before. I constantly need to check if you are there, and I get upset if I think you are ignoring me. Please don't. Even with all our differences, I love you till the end. I will push this relationship till death do us apart. Without you, I'm lost.

Sincerely, a CompSci Student's Final Project

  • master - Default branch name
  • reset - Undo local changes and overwrite with remote
  • status - List the files you've changed and those you still need to add or commit
  • .gitignore - Uses it to determine which files and directories to ignore
  • diff - View all the merge conflicts
  • push - Send changes to the master branch of your remote repository
  • repo - Place where the history of your work is stored
  • Readme.md - A marked down read me file that is HTML formatted
  • checkout - Switch from one branch to another
  • remote - Repository that is on a remote server rather than local
  • pull - Fetch and merge changes on the remote server to your working directory

Dear GitHub,
Where do I begin? You were there to read me like a book you checked out from the library since the first time I met you. Being so remote, yet so close, you're a mystery to me. You pulled me in like a black hole and never let go. I now count days till we hold hands and merge into one, so we can tell people what we have achieved. I know many others want you as well. I'm the luckiest person to be able to be yours and yours only.

Sincerely, Open Source Project Participant

  • merge - To merge a different branch into your active branch
  • release - An official release version of the project
  • tag - You can tag each repository release to refer back to the versions
  • contributor - People that have worked and committed to the repository
